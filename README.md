# mynet

**mynet** 是一个基于 C++ 开发的高性能、非阻塞网络底层库。它是 **Knight_RPC** 框架的核心通信引擎，采用了工业界成熟的 **Reactor** 设计模式，专为处理高并发、低延迟的网络服务而设计。

## ✨ 技术核心

* **事件驱动 (Event-Driven)**：底层深度封装 Linux **Epoll** 系统调用，支持海量文件描述符的高效监听。
* **主从 Reactor 模型 (Main-Sub Reactor)**：
    * **Main Reactor**：独立线程运行，专职处理 `accept` 事件，确保新连接接入无延迟。
    * **Sub Reactor 池**：多个 IO 线程并发运行，负责已连接 Socket 的数据读写，充分利用多核 CPU 性能。
* **应用层缓冲区 (Buffer)**：为每个连接设计了自动扩容的读写缓冲区，完美处理 TCP 粘包、断包以及写缓冲满的情况。
* **定时器管理**：内置基于小根堆或时间轮（根据你实际实现选一个）的定时器，支持高效清理死链接与超时请求。
* **线程安全**：基于 `std::mutex` 与 `std::condition_variable` 实现了健壮的线程池与任务队列。

---

## 🏗️ 模块划分

* **EventLoop**：事件循环核心，驱动整个系统的运行。
* **server**：监听套接字（listenfd）的封装，处理新连接。
* **group**:负责将server  accept的连接分发给eventloop线程
* **utils**：辅助功能模块集合，包括线程池，对象池，日志库，定时器等功能模块

---

## 🛠️ 性能表现

* **高并发**：在标准 Linux 服务器环境下，支持 **10w+** 并发连接稳定运行。
* **低延迟**：通过减少内存拷贝（Zero-copy 思想应用）与高效的线程调度，单次请求响应时间显著优于传统同步模型。
